# Litops Atelier — статический интернет-магазин

Готовый адаптивный прототип для публикации на GitHub Pages.

## Состав

- `index.html` — главная страница магазина.
- `style.css` — стили витрины и адаптивная версия.
- `script.js` — каталог, фильтры, поиск, избранное, корзина, формы и localStorage.
- `admin.html` — демонстрационная панель администратора.
- `admin.css` — стили панели администратора.
- `admin.js` — управление товарами и заявками, импорт и экспорт JSON.
- `assets/brand` — улучшенные версии логотипа.
- `assets/products` — оптимизированные фотографии товаров и исходные вертикальные изображения.

## Публикация на GitHub Pages

1. Создайте новый публичный репозиторий GitHub.
2. Загрузите **всё содержимое этой папки** в корень репозитория. Файл `index.html` должен лежать в корне.
3. Откройте `Settings` → `Pages`.
4. В разделе `Build and deployment` выберите `Deploy from a branch`.
5. Выберите ветку `main` и папку `/ (root)`, затем нажмите `Save`.
6. Через несколько минут GitHub покажет адрес опубликованного сайта.

## Как открыть админ-панель

Перейдите на `admin.html` или нажмите иконку управления в шапке сайта.

Изменения каталога сохраняются в `localStorage` текущего браузера и сразу отображаются на `index.html` в этом же браузере.

## Важное ограничение

GitHub Pages публикует только статические файлы. Поэтому текущая админ-панель является функциональным локальным прототипом:

- нет общей базы данных;
- изменения не синхронизируются между устройствами;
- покупатели не видят изменения, сделанные в вашем браузере;
- нет безопасной авторизации администратора;
- нет серверной обработки заказов и оплаты;
- изображения нельзя загружать на сервер из панели.

Для полноценного магазина следующая версия должна использовать backend, например Supabase. Тогда можно подключить авторизацию, общую базу товаров и заказов, загрузку изображений, Telegram-уведомления и онлайн-оплату.
